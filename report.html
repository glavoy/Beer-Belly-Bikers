Introduction

This website is about an informal group of bikers that call themselves the ‘Beer Belly Bikers’, of which I am a member.  Basically, we are a few expats living in East Africa and we all love to ride motorcycles.  And of course, we like to have a cold beer after a long days ride through hot, dusty roads.  So we get together once in a while to do rides around Uganda and Kenya.  I have tried to give a good description of the ‘Club’ and its members along with maps of previous rides and I also included a lot of images on the site

I used a ‘classic’ design for the site with a simple colour scheme..

State 3 things that have inspired you when creating your website. 

  • First and foremost, I thought it would be fun to create a website for our informal group.
  • Once I began working on it, I wanted to keep the design simple and minimal so that the website was not ‘busy’.
  • I wanted to make the site pleasurable so that other people would enjoy it.  I think the descriptions of the bikers as well as the descriptions about the past rides make the site enjoyable and interesting.

 

State 3 ways in which your site is accessible.

  • It uses a responsive design so that it can be viewed on any device.
  • All images have descriptive content for the ‘alt’ attribute.
  • It uses different variations of css code to accommodate all browsers, even older ones.

State 3 ways in which you have considered usability in your site.

  • There are links to all main pages from both the header and footer of each page.
  • The use of breadcrumbs ensures it is easy for the user to know where they are on the website at any time.
  • I kept the design simple and avoided using moving objects on the screen.  I did create one button that has continuous animation to attract user attention.  However, once the button is clicked, the animation stops.

 

State 3 things you had to learn/find out by yourself when creating your site, and describe

how you did that.

  • One of the first things I learned is that the order in which JavaScript is executed is really important.  I spent a lot of time troubleshooting my JavaScript code, only to find that a certain section of the code relied on another section of code, but it was in the incorrect order.
  • I struggled with the layout of the pages.  There are many different options available - Bootstrap, css grid, flexbox.  I tried several different methods using tutorials online and slowly began to understand the advantages of each of them.
  • I initially tried creating my website purely from scratch.  I tried writing my own css for responsive design, layout and animations.  I realized this was really a time waster since there are many free resources out there to help you get up and running quickly.  I then decided to use Bootstrap to speed up development.  Basically, there is no need to reinvent the wheel.

What aspect(s) of your work do you think worked well and why?

I think the simple design and the colour scheme look pleasing.  Personally, I like the look and feel of the site, but of course I am probably biased since I designed it.

I also liked the code for the photo gallery, which I wrote in JavaScript completely from scratch.

 

 

What aspect(s) of your work could be improved, and how might you do things differently

another time?

I realized it is really important to create wireframes and/or mockups before you jump into coding.  Although I did create rough wireframes of the pages before I began, I found myself drifting away from the original design when I began coding.  After some time, I went back and re-did all of the wireframe layouts with new ideas I had.  I realized that one you have the layout on paper, coding becomes a lot easier.

Also, I didn't use a templating language like Handlebars. I think something like that could have been useful on the 'members' pages since they all have the lame layout and only the content changes.
